## 概要

Playwright Test Agents（playwright-test-planner / playwright-test-generator）を使い、
テスト計画生成とコード生成を「入力条件の異なる3パターン」で比較する実験。

---

## パターン定義

| パターン                  | 入力                              | アプリ探索                   | 説明                                      |
| ------------------------- | --------------------------------- | ---------------------------- | ----------------------------------------- |
| **A: 設計書のみ**         | 設計書5ファイル                   | なし                         | 実装前にテスト設計を行うケース            |
| **B: 設計書＋コード参照** | 設計書5ファイル＋既存テストコード | なし（コード参照で代替）     | `planner_setup_page` 未動作のため代替実施 |
| **C: 設計書＋アプリ探索** | 設計書5ファイル                   | あり（`planner_setup_page`） | 実装後にテスト設計・実装するケース        |

すべてのパターンで `docs/test-spec.md` は**提供しない**（比較対象のため）。

## フェーズ1: テスト計画生成（Planner）

### 結果

| パターン | ツール                                                    | シナリオ数 | 完了    |
| -------- | --------------------------------------------------------- | ---------- | ------- |
| A        | `playwright-test-planner`（設計書のみ）                   | 116件      | ✅ #114 |
| B        | `playwright-test-planner`（設計書＋コード参照）           | 73件       | ✅ #115 |
| C        | `playwright-test-planner`（設計書＋`planner_setup_page`） | 101件      | ✅ #116 |

### 発見事項

**MCP サーバー名衝突（Pattern B → Pattern C 移行の原因）**

Pattern B 実施時に `.mcp.json` のサーバー名 `playwright-test` が既存サーバーと衝突し、
`planner_setup_page` が利用不可になった。修正方法: サーバー名を `playwright-test-runner` に変更。

**サブエージェントへの MCP 接続が伝播しない**

`Agent` ツールで起動したサブエージェントは、メインセッションの MCP サーバー接続を継承しない。
エージェント定義の `tools:` フィールドに MCP ツールを指定しても、
実際の MCP サーバーへの接続がなければ使用不可。

→ Pattern C の Planner は「既存の spec ファイルを参照」する形で動作し、
ブラウザ直接起動は行わなかった（アプリ探索は限定的）。

### fixme 理由（ET071）

E511（最後のMANAGER削除不可）より E510（自己削除不可）が先に返される。
テスト自体の設計問題のため `test.fixme()` でスキップ。

## 備考: MCP サーバー関連の知見

### サブエージェントへの MCP 接続非伝播

`Agent` ツールで起動したサブエージェントは MCP サーバーへの接続を継承しない。
エージェント定義の `tools:` フィールドに MCP ツール名を記載しても、
実際の接続がなければ「No such tool available」エラーとなる。

**回避策**: MCP ツールが必要な操作はメインセッションで直接実行する。

### `generator_write_test` の独立性

`generator_write_test` は `generator_setup_page` や `generator_read_log` とは独立して使用可能。
ブラウザ操作なしにコードをファイルに書き込める。

ただし、ブラウザで動作確認していないため、実際のアプリの UI と
テストコードの間にギャップが生じる（→ 初回失敗率: 20/116 = 17%）。

### `generator_setup_page` のシードファイル要件

シードファイルは `page` フィクスチャを使うテストである必要がある。
`async () =>` （フィクスチャなし）のテストは `test_debug` モードで動作しないためエラーになる。

→ 回避策: `e2e/seed-generator.spec.ts` を作成（`page` フィクスチャを受け取る形式）。
