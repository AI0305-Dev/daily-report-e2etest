# E2Eテスト作成方法の比較実験

## 目的

PlaywrightのUI（codegen）を使った手動作成と、AIを活用した自動生成を比較し、  
**テストコード作成におけるAI活用の有用性**を工数・品質の観点から示す。

---

## ディレクトリ構成

```
e2e/comparison/
  README.md            ← このファイル（実験計画・記録）
  manual/              ← UIから手動で作成したテスト
  ai/                  ← AI支援で作成したテスト
  ai-reviewed/         ← 手動コードをAIがレビュー・修正したテスト
```

## 実験手順

### 手動作成（manual/）

1. Playwrightのcodegenを起動する
   ```bash
   npx playwright codegen http://localhost:3000
   ```
2. ブラウザ上で操作を記録し、生成されたコードを `manual/` に保存する
3. 必要に応じてコードを調整し、テストが安定してpassすることを確認する
4. **作業時間・修正回数を下の記録欄に記入する**

### AI生成（ai/）

1. テスト仕様（テストID・手順・期待結果）をAIに提示する
2. 生成されたコードを `ai/` に保存する
3. テストを実行し、必要なら修正をAIに依頼する
4. **作業時間・修正回数を下の記録欄に記入する**

## 最終まとめ（実験完了後に記入）

| 指標         | 手動 | AI生成 | AIレビュー修正 |
| ------------ | ---- | ------ | -------------- |
| 作成時間合計 | 4.0h | 約6分  | 約5分          |
| 平均行数     | 55行 | 26行   | 28行           |
| 修正回数合計 | 11回 | 1回    | 0回            |

**総合評価**

- AI生成・AIレビュー修正ともに手動の約1/40〜1/50の時間で完了
- AIレビュー修正版は手動コードの問題点（重複・不要操作）を解消しつつ、カバレッジを維持
- AI生成は初回でカバレッジが不足する場合があるため、仕様との照合レビューが必要
- 複雑なフローほど手動との差が顕著

## 全テストへの外挿試算

本プロジェクトのテスト仕様書（`docs/test-spec.md`）に定義されたE2Eテストは **62件**（ET001〜ET137）。  
今回の比較実験3件の平均値をもとに全件作成した場合の工数を試算する。

### 計算根拠

| 指標               | 3件の実測値 | 1件あたり平均 |
| ------------------ | ----------- | ------------- |
| 手動作成時間       | 4.0h        | 約80分        |
| AI生成時間         | 約6分       | 約2分         |
| AIレビュー修正時間 | 約5分       | 約1.7分       |

### 62件全体への外挿

| 手法                   | 試算工数               | 備考                         |
| ---------------------- | ---------------------- | ---------------------------- |
| 手動                   | 約83時間（約10営業日） | コーディング・デバッグ含む   |
| AI生成                 | 約2時間                | 仕様照合レビュー時間を含まず |
| AI生成（レビュー込み） | 約3〜4時間             | 仕様との照合・修正を含む想定 |
| AIレビュー修正         | 約1.8時間              | 既存手動コードがある場合のみ |

### インパクトまとめ

- 手動 → AI生成: **約40倍の効率化**（83時間 → 約2時間）
- E2E整備を「後回しにする理由がなくなる」レベルの差
- AI生成は仕様との照合コストを加味しても、手動の **1/10未満** に収まる見込み
