**バージョン**: 1.0

## 1. テスト方針

| 種別             | ツール     | 対象                                          |
| ---------------- | ---------- | --------------------------------------------- |
| 単体テスト（UT） | Vitest     | Zodバリデーションスキーマ・ユーティリティ関数 |
| 統合テスト（IT） | Vitest     | APIエンドポイント（テスト用DBに対して実行）   |
| E2Eテスト（ET）  | Playwright | 主要ユーザーフロー（ブラウザ操作）            |

### テストID体系

| プレフィックス | 種別       |
| -------------- | ---------- |
| UT             | 単体テスト |
| IT             | 統合テスト |
| ET             | E2Eテスト  |

## 3. 統合テスト（Vitest）

テスト用DBを使用し、APIエンドポイントをHTTPリクエストで検証する。  
各テスト前にDBをシードデータで初期化する。

### シードデータ（最小構成）

各統合テストの前提として、以下のデータをDBに投入する。

#### ユーザー

| 識別子   | 氏名     | メール            | ロール  | isAdmin | 上長     |
| -------- | -------- | ----------------- | ------- | ------- | -------- |
| MANAGER1 | 鈴木部長 | manager1@test.com | MANAGER | true    | -        |
| MANAGER2 | 田中部長 | manager2@test.com | MANAGER | false   | -        |
| SALES1   | 山田太郎 | sales1@test.com   | SALES   | false   | MANAGER1 |
| SALES2   | 佐藤次郎 | sales2@test.com   | SALES   | false   | MANAGER1 |

#### 顧客

| 識別子       | 顧客名       | isDeleted |
| ------------ | ------------ | --------- |
| CUSTOMER1    | テスト顧客A  | false     |
| CUSTOMER2    | テスト顧客B  | false     |
| CUSTOMER_DEL | 削除済み顧客 | true      |

#### 日報・コメント

| 識別子              | userId | 日付     | status    |
| ------------------- | ------ | -------- | --------- |
| REPORT_DRAFT        | SALES1 | 当日-4日 | DRAFT     |
| REPORT_SUBMITTED    | SALES1 | 当日-3日 | SUBMITTED |
| REPORT_REJECTED     | SALES1 | 当日-2日 | REJECTED  |
| REPORT_COMPLETED    | SALES1 | 当日-1日 | COMPLETED |
| REPORT_S2_SUBMITTED | SALES2 | 当日-1日 | SUBMITTED |

- REPORT_SUBMITTED には CUSTOMER1 への訪問記録（content="テスト訪問"）が1件
- REPORT_COMPLETED には MANAGER1 からの GENERAL コメント（body="承認済みコメント"）が1件
