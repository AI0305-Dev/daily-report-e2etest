# 承認・差し戻しフロー テスト計画

**Seed:** `e2e/seed.spec.ts`

---

## ET040: MANAGER が提出済み日報を承認する

**前提条件:** `manager1@test.com` でログイン済み。山田太郎の SUBMITTED 日報（当日-3）が存在する。

**手順:**

1. `/manager/reports` を開く
2. 山田太郎の提出済み日報行をクリックする
3. 日報詳細画面で「承認」ボタンをクリックする
4. 確認ダイアログが表示されることを確認する
5. ダイアログで「OK」ボタンをクリックする

**期待結果:** `/manager/reports` にリダイレクトされる。承認した日報のステータスが「完了」になっている

---

## ET041: MANAGER が提出済み日報を差し戻す

**前提条件:** `manager1@test.com` でログイン済み。山田太郎の SUBMITTED 日報（当日-3）が存在する。

**手順:**

1. `/manager/reports` を開く
2. 山田太郎の提出済み日報行をクリックする
3. 「差し戻し」ボタンをクリックする
4. 確認ダイアログが表示されることを確認する
5. 差し戻し理由入力欄に「訪問内容が不十分です」と入力する
6. ダイアログで「OK」ボタンをクリックする

**期待結果:** `/manager/reports` にリダイレクトされる。対象日報のステータスが「差し戻し」になっている

## ET042: 差し戻し後に SALES が修正・再提出できる

**前提条件:** ET041 の実行後。`sales1@test.com` でログイン済み。

**手順:**

1. `/reports` を開く
2. 差し戻された日報行（「差し戻し」ステータス）をクリックする
3. 「編集」ボタンをクリックする
4. Problem 欄に「修正した内容」を追記する
5. 「提出」ボタンをクリックする

**期待結果:** 日報詳細画面でステータスが「提出済」と表示される
